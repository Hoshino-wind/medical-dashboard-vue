# 医疗设备大屏“全息仪器系统”视觉改造规格

- 日期：2026-07-10
- 状态：已确认
- 选定方向：B — 全息仪器系统

## 1. 目标

在保留当前 3×3 九宫格、深蓝科技基调和 2.5D 图表资产的前提下，将现有“多个科技模块的集合”提升为一套统一、精密、高端的医疗全息仪器系统。

完成后的大屏应具备以下特征：

- 第一眼明确看到顶部标题、中央核心列、左右辅助仪器的层级。
- 2.5D 图表使用一致的透视、光源、底座、厚度和运动语言。
- 炫酷感来自材质、光影、纵深与节奏，而不是所有元素同时高亮。
- 配置页与大屏属于同一产品视觉体系，同时保持编辑器的可操作性。
- 1920×1080 桌面大屏接近稳定 60fps。

## 2. 范围

### 2.1 本次包含

- 全局视觉层级与色彩职责重构。
- 面板玻璃、边框、切角、阴影与发光系统统一。
- 中央能量脊柱及九宫格整体联系感。
- 环形仪表、全息底座、3D 饼图、2.5D 柱状图和折线图视觉统一。
- 九个大屏模块的视觉精修。
- 整屏入场、常驻和数据变化动画编排。
- 配置页组件池、布局区、属性区和预览区的视觉与交互精修。
- 键盘焦点、减少动态效果和关键状态非纯色表达。

### 2.2 本次不包含

- 数据内容、指标口径、空数据或 mock 数据调整。
- 后端接口、数据刷新协议和业务逻辑调整。
- 手机端和窄屏响应式重构。
- 路由结构调整。
- 新增页面。
- 改变当前 3×3 与 2×3 布局能力。

## 3. 设计方向契约

### 3.1 表面类型

产品型技术仪器界面。它是用于展示和管理医疗设备状态的高端大屏，不是普通管理后台，也不是游戏 HUD。

### 3.2 风格家族

深海医疗精密仪器：深蓝黑环境、冷白读数、青蓝结构光、少量蓝紫反射、真实的仪器层级与纵深。

### 3.3 视觉感受

- 精密
- 克制
- 高能

### 3.4 反向约束

界面不得呈现以下观感：

- 夜店霓虹：大面积高饱和发光和持续闪烁。
- 廉价科幻：无业务关系的装饰线、随机粒子和伪技术标签。
- 游戏 HUD：过多尖角、瞄准器、战术符号和军事化语言。

### 3.5 标志性形式

一条从顶部标题核心向下贯穿中央列的“中央能量脊柱”。它通过极弱流动光线连接维修、巡检和保养三个中央模块，使九宫格形成一个完整仪器系统。

### 3.6 克制规则

- 常态高亮面积不超过画面面积的约 10%。
- 同一时刻最多出现一个主高亮动画和两个辅助动画。
- 单个模块最多使用两种主数据色。
- 保留现有循环动画；如果画面过于繁忙，优先降低循环动画亮度、拉长周期并错开相位，不直接移除循环生命感。

## 4. 保留、调整与淘汰

### 4.1 保留

- 3×3 九宫格总体结构。
- 顶部医院、品牌、系统标题三段式构图。
- 深蓝科技主题和现有主题切换能力。
- 环形仪表、全息底座、3D 饼图和 2.5D 柱状图。
- 星点、流光与科技框架，但重新控制强度和节奏。
- 当前模块标题、数据结构和组件边界。

### 4.2 调整

- 九个同权面板调整为中央主、左右辅的三级层级。
- 统一 2.5D 图表透视和光照。
- 图表、表格和底部统计条使用同一仪器材质。
- 统一字体、数字、单位、状态和图例语法。
- 配置页从彩色块编辑器调整为全息布局编辑器。
- 保留各模块现有循环动画，并调整为“整屏有序启动、循环效果分级、相位错峰”的统一节奏。

### 4.3 淘汰

- 所有面板同强度持续外发光。
- 无状态意义的大面积纯蓝、纯绿卡片。
- 图表之间互不一致的透视和材质。
- 同屏多个同相位、同亮度的高能循环动画；循环动画本身必须保留，但需要建立主次与错峰。
- 只依赖颜色表达模块类型或状态。

## 5. 整屏构图与视觉层级

### 5.1 视觉路径

五秒观看路径固定为：

1. 顶部系统标题。
2. 中央能量脊柱。
3. 中央列的工单与状态仪器。
4. 左侧可用率仪表阵列。
5. 右侧趋势与统计图表。

### 5.2 四层视觉模型

#### 环境背景层（约 65%）

- 深海蓝黑纵向渐变。
- 星点保留，但高亮星点密度下降。
- 极弱纵深网格和雾化环境光。
- 背景元素不得与数据形成同等亮度。

#### 结构框架层（约 20%）

- 九宫格边界、中央脊柱、面板连接线和角部刻度。
- 使用低亮青蓝结构光。
- 中央列结构亮度比左右列高约一级，但不扩大占位。

#### 数据仪器层（约 10%）

- 环形仪表、柱状图、饼图、折线、表格主数据和关键数字。
- 冷白数字优先于彩色装饰。
- 数据主体与背景至少形成明显明度差。

#### 高能状态层（约 5%）

- 标题流光、当前扫描位置、状态切换、数据更新和告警脉冲。
- 只在事件发生或低频节奏点出现。

## 6. 色彩系统

现有主题变量继续作为基础，通过职责化 token 扩展，而不是在组件内新增随机颜色。

| 色彩角色 | 推荐范围 | 用途 |
| --- | --- | --- |
| 环境底色 | `#020814`–`#041A2E` | 页面背景、深层空间 |
| 结构青蓝 | `#16B8D9`、`#38BDF8` | 面板边缘、坐标、底座结构 |
| 电蓝 | `#4F8CFF` | 第二层结构、辅助数据 |
| 冷白 | `#EEF8FF` | 标题、核心数字、能量亮芯 |
| 蓝紫 | `#8B7CF6`–`#A679FF` | 第二数据系列、维保类反射 |
| 正常绿 | `#2DD48F` | 完成、正常、可用 |
| 警示琥珀 | `#F4C152` | 待处理、临近状态 |
| 危险红 | `#F06578` | 超时、故障、危险 |

约束：

- 冷白只用于阅读焦点和亮芯，不作为大面积背景。
- 紫色只作为第二数据系列和局部环境反射。
- 状态色必须同时配合文字、图形或能量点形态，不能只靠颜色。

## 7. 面板材质系统

所有模块使用“精密仪器玻璃”材质：

1. 外层为低亮渐变边框。
2. 内层为极细冷白高光线。
3. 四角使用短机械切角或能量刻度。
4. 面板顶部略亮，底部略暗，形成厚度。
5. 底边带有极弱环境反射。
6. 只有悬停、聚焦、数据更新或告警时增强外发光。

面板等级：

- A 级：中央列模块；结构最完整，边缘亮度最高。
- B 级：左右可用率与总览仪表；保持明显纵深。
- C 级：普通趋势和统计模块；边框更安静，让图表成为主角。

## 8. 2.5D 仪器规范

### 8.1 统一透视

- 建议俯视角：18°–22°。
- 圆形顶部椭圆压缩比：约 0.32–0.38。
- 立体厚度：视觉直径的约 6%–10%。
- 主光源：左上方冷白光。
- 环境反射：右下方蓝紫光。
- 同类图表不得使用相反光源或明显不同俯视角。

### 8.2 三层全息底座

1. 外层定位环：暗青结构光，极慢旋转。
2. 中层能量环：负责低频呼吸和更新波纹。
3. 内层投影盘：承接仪表、饼图或主数据对象。

底座可根据模块改变尺寸和细节，但透视、光照、材质和运动方向必须一致。

### 8.3 图表家族

- 仪表：冷白数字、青蓝进度、三层底座。
- 饼图：同一厚度、光源、切片间隙和投影盘。
- 柱状图：同一立体厚度、顶面光照、地面网格和倒影。
- 折线图：低亮坐标、能量轨迹、当前点脉冲和短拖尾。

## 9. 字体与数字规范

以 1920×1080 基准：

- 系统标题：30–34px，冷白色，较宽字距。
- 模块标题：16–18px，中等字重，冷白或浅青。
- 核心数据：32–44px，等宽数字，启用 `tabular-nums`。
- 表格正文：14–16px，字重低于标题。
- 单位：主数字尺寸的 40%–50%，与数字基线对齐。

规则：

- 中文继续使用当前系统字体栈。
- 数字优先使用系统可用的仪表感字体回退栈，不引入阻塞加载的远程字体。
- 不给所有文字加发光；发光仅用于标题边缘、核心数字和事件状态。

## 10. 九个模块规格

### 10.1 设备总览

- 中央仪表环视觉直径增加约 10%。
- 四个指标卡向中央形成轻微放射关系，并以极细连接线关联。
- 中心数字冷白，外圈青蓝渐变。
- 底座提供从内向外扩散的低频扫描波。
- 0 值保持待机呼吸，不改变数据或展示伪数据。

### 10.2 维修工单

- 调整为全息数据终端材质。
- 表头使用暗金属蓝底。
- 当前滚动行经过时出现短暂边缘扫描线。
- 状态增加能量点或小图形，不仅使用文字颜色。
- 底部统计指标合并为连续仪器坞。

### 10.3 保修统计

- 统一柱体左上高光和右侧阴影。
- 增加透视地面网格和低透明倒影。
- 入场时柱体从底部能量环中升起。
- 只有当前或最高柱保持低频脉冲。
- 图例收敛为右上角紧凑指示器。

### 10.4 生命支持设备可用率

- 三个仪表组成统一设备阵列。
- 三个底座通过极弱轨迹连接。
- 底座呼吸错峰，不同时高亮。
- 中间仪表轻微前移，产生纵深。
- 环形扫描方向一致。

### 10.5 巡检工单

- 左侧表格与右侧完成率仪器使用垂直能量隔板连接。
- 右侧饼图嵌入仪器槽，不作为独立悬浮图形。
- 底部统计合并为连续控制台底栏。
- 顶边接入中央能量脊柱的下行光线。

### 10.6 保养统计

- 折线增加短能量拖尾。
- 只让当前数据点低频脉冲。
- 坐标轴降亮，数据线提亮。
- 图表底部提供极弱镜面投影。
- 数据更新时播放一次明显的轨迹推进；常态保留低亮度循环流光，周期拉长并与其他模块错峰。

### 10.7 超声设备可用率

- 与生命支持设备共用仪器家族。
- 主色偏青蓝，局部反射使用蓝紫以形成类别差异。
- 不改变底座材质、透视和光源。
- 紫色不覆盖普通文字和全部结构。

### 10.8 保养工单

- 健康状态饼图调整为嵌入式诊断仪器。
- 圆环中心增加诊断核心。
- 状态变化使用沿圆周传递的增强光线；常态保留低速整体循环和底座旋转，状态变化时临时提高能量强度。
- 左侧表格、右侧诊断仪和底部状态条形成同一控制台。

### 10.9 巡检统计

- 与保养统计共用时间轨迹系统。
- 使用青绿色作为主系列色。
- 末端数据点低频呼吸。
- 折线绘制完成后迅速降低环境光，避免持续抢眼。

## 11. 动效系统

### 11.1 整屏入场

| 时间 | 动作 |
| --- | --- |
| 0–300ms | 背景网格和星点显现 |
| 200–700ms | 顶部框架从两侧向中央汇聚 |
| 500–900ms | 标题核心点亮 |
| 650–1300ms | 中央列由上至下启动 |
| 850–1500ms | 左右面板向外展开 |
| 1100–2000ms | 仪表、柱体和折线依次生长 |
| 2000ms 后 | 进入低频待机状态 |

### 11.2 常驻动画

- 保留现有循环动画，不以静态化作为优化目标。
- 标题流光：8–10 秒一次。
- 全息底座呼吸：6–8 秒一次，各模块错峰。
- 背景扫描：12–16 秒一次。
- 中央能量脊柱：低亮度缓慢流动，不产生强闪烁。
- 面板边缘、仪表扫描、状态微光和图表轨迹可持续循环，但必须使用不同周期与相位。
- 循环动画按主、辅、环境三级分配亮度；允许多个循环同时存在，但同一时刻最多一个主高亮和两个辅助高亮。

### 11.3 数据变化动画

- 数字变化：约 450ms。
- 仪表环推进：约 700ms。
- 柱状图生长：约 900ms。
- 状态切换：旧状态先收暗，再点亮新状态。
- 动画结束后回到稳定低亮状态。

### 11.4 减少动态效果

当系统开启 `prefers-reduced-motion` 时：

- 以下降级仅响应用户主动设置的减少动态效果偏好，不改变默认模式下保留循环动画的要求。
- 取消持续旋转、呼吸和扫光。
- 入场改为短淡入。
- 数据变化保留瞬时状态更新或极短透明度过渡。
- 不影响内容可见性和操作反馈。

## 12. 配置页规格

### 12.1 组件池

- 彩色纯色卡改为深色半透明仪器卡。
- 类型色仅用于边缘光和类型标识。
- 每张卡包含类型、名称和微型图表预览。
- 卡片悬停轻微抬升，聚焦使用清晰冷白描边。

### 12.2 布局区

- 九宫格改为带纵深的仪器安装槽。
- 空槽显示低亮十字定位线。
- 拖起模块时卡片浮起并增强阴影。
- 合法槽位显示青色吸附光。
- 非法槽位显示琥珀色阻挡框并保留文本反馈。
- 放置完成播放一次短扩散波。
- 移除使用明确按钮，不再通过点击整个槽位直接删除。

### 12.3 属性区

- 2×3、3×3 使用结构缩略图选择。
- 主题选择使用材质色板，展示背景、结构光和数据光。
- 选中项使用双层描边和小型确认图标。
- 配置数量使用紧凑型进度仪表，并保留文本数值。

### 12.4 预览区

- 提高预览区域视觉权重。
- 增加明确的全屏预览入口。
- 主题切换使用 300–450ms 平滑过渡。
- 预览与配置区共享同一材质和光照系统。

### 12.5 保存与破坏性操作

- 本地自动持久化机制保持不变。
- 界面文案必须清楚区分“自动保存”和“发布演示”。
- 清空和重置使用危险层级样式，并提供确认或可撤销反馈。

## 13. 技术落点

优先复用现有 Vue、CSS token、ECharts、Three.js 和组件结构，不引入新的大型 UI 框架。

主要修改区域：

- `src/styles/tokens.css`：职责化颜色、材质、深度、动效 token。
- `src/styles/layout.css`：中央能量脊柱和全局层级。
- `src/styles/panel.css`：A/B/C 级仪器面板。
- `src/styles/header.css`：标题能量核心与启动时序。
- `src/styles/charts.css`、`src/styles/rings.css`：图表与仪表家族。
- `src/styles/modules.css`、`src/styles/tables.css`：模块和数据终端细节。
- `src/styles/config.css`：全息布局编辑器。
- `src/components/visual/*`：统一底座、光照和运动参数。
- `src/components/charts/*`：统一柱状图、饼图和折线图视觉语法。
- `src/components/modules/*`：九个模块的组合精修。

组件继续通过现有主题变量读取颜色。新增 token 必须有明确职责并能被多个组件复用，禁止为单个装饰效果创建无复用的一次性变量。

## 14. 性能约束

- 目标环境：1920×1080 桌面浏览器。
- 目标：接近稳定 60fps；持续动画单帧预算约 16.7ms。
- 大面积模糊、阴影和混合模式不得同时叠加多层。
- 常驻动画优先使用 `transform` 和 `opacity`。
- ECharts 和 Three.js 动画必须避免重复启动和无意义重绘。
- WebGL、Canvas 和 CSS 粒子效果不重复承担同一视觉职责。
- 动画错峰，避免九个模块同一时间进行高能变化。

## 15. 可访问性约束

- 所有交互控件提供可见的 `focus-visible` 状态。
- 状态和模块类型不能只通过颜色表达。
- 动画遵循 `prefers-reduced-motion`。
- 装饰图形保持 `aria-hidden`，业务图表保留可理解的文本描述。
- 聚焦、选中、拖拽合法和拖拽非法状态均提供非纯视觉反馈。

## 16. 实施顺序

1. 建立职责化 token、面板等级和数字字体系统。
2. 统一 2.5D 透视、光源和三层全息底座。
3. 建立中央能量脊柱和整屏层级。
4. 优先改造设备总览、巡检工单和保养工单三个核心模块。
5. 改造其余仪表、柱状图、折线图和表格。
6. 编排入场、常驻和数据变化动画。
7. 重构配置页视觉与拖拽反馈。
8. 运行视觉、交互、性能、减少动态效果和现有自动化测试。

## 17. 验收标准

- 缩小为缩略图后仍能辨认“标题—中央核心—左右辅助”的层级。
- 九个面板不再具有完全相同的视觉重量。
- 所有 2.5D 图形使用一致的透视、光源和底座语言。
- 同一时刻高亮运动元素不超过三个。
- 单模块主数据色不超过两种。
- 空数据保持当前真实值，只改变仪器待机表现。
- 配置页与大屏明显属于同一视觉系统。
- 键盘可以识别当前焦点，减少动态效果模式下内容和交互完整。
- 1920×1080 下无明显掉帧、裁切、文字重叠或图表溢出。
- 现有单元测试、类型检查、构建和 lint 通过，或明确记录与本次无关的既有失败。

## 18. 实施边界

本规格锁定以下决策：

- 继续使用九宫格，不改为中央大屏加卫星卡片的结构。
- 继续使用深蓝高端科技方向，不改为浅色医疗后台。
- 强化而非删除 2.5D。
- 数据保持原样。
- 暂不处理移动端。

工程实现可以在不改变以上决策的情况下调整具体像素、动画时长和渲染方式，以满足视觉一致性和性能要求。任何需要改变布局结构、主题方向、数据展示或响应式范围的调整都需要重新确认。
